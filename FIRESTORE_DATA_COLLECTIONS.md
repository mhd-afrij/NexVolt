# Firestore Data Collections

This document describes the Firestore collections used by the current NexVolt app codebase and the fields written or read by the app.

## Overview

The app currently uses one root Firestore collection and subcollections beneath it:

- `nexvolt-db/bootstrap`
- `nexvolt-db/bootstrap/users`
- `nexvolt-db/bootstrap/vehicles`
- `nexvolt-db/bootstrap/stations`
- `nexvolt-db/bootstrap/charging_sessions`

The app also supports a local in-memory fallback when Firebase is unavailable. In that mode, the same repository methods work, but no Firestore writes happen.

## `nexvolt-db`

This is a bootstrap collection used at app startup.

### Document

- `nexvolt-db/bootstrap`

### Fields

- `createdAt` - server timestamp
- `source` - string, currently `app-startup`

### Purpose

This collection is created on startup to ensure Firestore is reachable and initialized.

## `nexvolt-db/bootstrap/users`

This collection stores the authenticated user's profile document.

### Document ID

- The Firebase Auth user ID (`uid`)

### Fields written by the app

- `userId` - the same value as the document ID
- `updatedAt` - server timestamp
- `name` - user display name
- `email` - email address
- `homeCity` - home city name
- `homeLatitude` - latitude for the user's home location
- `homeLongitude` - longitude for the user's home location

### Read behavior

The app listens to the current user's document and uses it as the profile source.

## `nexvolt-db/bootstrap/vehicles`

This collection stores the vehicle selected or saved by the user.

### Document ID

- Auto-generated by Firestore

### Fields written by the app

- `userId` - owner Firebase Auth UID
- `vehicle` - selected vehicle name from the list, for example `Tesla Model 3`
- `vehicleType` - type such as `Car` or `Bike`
- `model` - vehicle model or display name
- `plate` - display label used by the UI
- `batteryPercent` - numeric value currently derived from the digits in `batteryVoltage`
- `batteryVoltage` - selected battery voltage string
- `connectorType` - selected connector string
- `battery` - legacy or fallback battery field
- `connector` - legacy or fallback connector field
- `createdAt` - server timestamp

### Example documents

```json
{
  "userId": "abc123",
  "vehicle": "Tesla Model 3",
  "vehicleType": "Car",
  "model": "Tesla Model 3",
  "plate": "Car",
  "batteryPercent": 350,
  "batteryVoltage": "350-400V",
  "connectorType": "Type 2 / CCS",
  "battery": "350-400V",
  "connector": "Type 2 / CCS",
  "createdAt": "server timestamp"
}
```

### Read behavior

The app queries `vehicles` by `userId` and orders by `createdAt` descending.

## `nexvolt-db/bootstrap/stations`

This collection stores charging station data.

### Document ID

- Usually the station ID

### Fields read by the app

- `name` - station name
- `address` - station address or description
- `distanceKm` - distance from the current location
- `latitude` - station latitude
- `longitude` - station longitude
- `availableSlots` - optional live availability count
- `totalSlots` - optional total slot count
- `status` - optional live status string

### Fields written by the app

- `availableSlots` - live availability count
- `totalSlots` - total number of slots
- `status` - station status
- `updatedAt` - server timestamp

### Purpose

The app reads station documents for list and detail views and updates availability for admin-style flows.

## `nexvolt-db/bootstrap/charging_sessions`

This collection stores charging activity history.

### Document ID

- Auto-generated by Firestore

### Fields written by the app

- `userId` - Firebase Auth UID
- `stationName` - name of the charging station
- `energyKwh` - energy used in kilowatt-hours
- `timestamp` - server timestamp

### Read behavior

The app queries charging sessions by `userId` and orders by `timestamp` descending.

## Current write paths in code

- Startup bootstrap: `nexvolt-db/bootstrap`
- Profile creation and updates: `nexvolt-db/bootstrap/users/{uid}`
- Vehicle registration: `nexvolt-db/bootstrap/vehicles`
- Station availability updates: `nexvolt-db/bootstrap/stations/{stationId}`
- Charging activity logging: `nexvolt-db/bootstrap/charging_sessions`

## Notes

- The app falls back to local in-memory data when Firebase initialization fails.
- If you change the Firestore schema, update both the Firestore write methods and the model parsing code.
- The UI currently reads vehicle display values from `vehicle`, `model`, `plate`, `batteryVoltage`, `connectorType`, and `batteryPercent`.
